- (decision) AD paradigm: hexagonal (ports and adapters) Rust core + unidirectional view-model projection. keeper-core owns domain + matrix orchestration behind ports; tauri shell is an adapter; React is a pure renderer of streamed VMs; all mutations round-trip through Rust commands. Prevents: two sources of truth, matrix logic leaking to TS
- (decision) AD crate layout: convert src-tauri to cargo workspace: keeper-core (NO tauri dependency; modules: accounts, auth, sync, inbox, timeline, send, drafts, archive, bridges, signals(incognito), notify, keychain, settings, vm(IPC DTOs)) and keeper (tauri shell: commands, channels, keeper-media protocol, plugins, tray). Dependency direction: keeper -> keeper-core -> matrix-sdk; keeper-core never imports tauri. Prevents: core tauri-bound, blocks mobile reuse and pure-Rust testing
- (decision) AD type sharing: ts-rs 12 derives on every IPC DTO (serde-compat), exported to src/lib/ipc/gen via cargo test export; hand-written thin typed wrappers (invoke + Channel) in src/lib/ipc. Rejected tauri-specta: still release-candidate; stacking an RC binding framework on matrix-sdk 0.x churn is double churn; ts-rs is stable, zero runtime coupling. CI diff-check keeps bindings fresh
- (decision) AD frontend state: zustand 5 vanilla stores created outside React, one per stream domain (accounts, inbox, timeline per open room, bridges, send/outbox, drafts/approval, settings); channel handlers apply VectorDiff batches imperatively off-render; components subscribe via selectors. Rejected TanStack Query (pull/cache model mismatches push streams) and bare useReducer (state must outlive components). Stores are mirrors only; no derived truth in JS
- (decision) AD IPC contract: commands named domain_verb snake_case; every fallible command returns Result<T, IpcError{code stable enum, message, account_id?, retriable bool}>; subscriptions are commands taking Channel<Batch> and returning subscription id; every stream opens with a reset/snapshot batch then VectorDiff ops so resubscribe is always safe; events (low-frequency) named keeper://kebab-case. JSON payloads; media never over IPC
- (decision) AD storage layout: app data dir = accounts/<account_id>/sdk/ (matrix-sdk-sqlite state+crypto+event-cache+media cache; deleted on logout), keeper.db (app DB: drafts, outbox, settings, bridge registry, account registry), archive.db (Local Archive: normalized events for ALL accounts keyed by account_id + FTS5; never touched by logout; per-account delete = explicit DELETE). One archive DB, not per-account files: SQLite ATTACH limit (default 10) breaks unlimited accounts for cross-account FTS. Keychain via keyring crate (apple-native store): access/refresh tokens, recovery keys, store passphrases; service dev.tgorka.keeper
- (decision) AD archive ingestion: archiver task per account subscribes to decrypted sync/timeline events post-decryption in Rust and appends normalized rows (event_id, account_id, room_id, sender, origin_ts, type, content_json, edit-version chain, redaction flag, media metadata). Edits append versions, redactions mark not erase (honor-remote-deletions setting governs retention only; timeline VIEW always honors redaction). Single serialized writer per DB, WAL mode (NFR-8). Export JSON/Markdown reads archive.db only, background task with progress
- (decision) AD FTS: SQLite FTS5 with trigram tokenizer (case-insensitive) — handles CJK without word segmentation and accelerates LIKE/GLOB for sub-3-char queries; external-content table over archive messages, incremental index on ingest. Closes PRD open question 6. 200ms/100k bar validated in CI perf test
- (decision) AD undo-send: approved sends INSERT into outbox (keeper.db) with dispatch_at = approval_time + window (0-60s, default 10); tokio scheduler moves elapsed rows into that account SendQueue; cancel during window deletes row and restores draft text to composer/drafts. Crash/offline: on restart elapsed rows dispatch immediately (PRD FR-46 assumption), unelapsed resume countdown. Held messages projected into timeline VM as held state with countdown by send module. Post-dispatch delete = matrix redaction. FR-41 enforcement: the ONLY path into SendQueue is send::submit(draft_or_text, trigger) where trigger in {ComposerSend, ApprovalPaneApprove}; no other public API dispatches
- (decision) AD incognito: signals module in keeper-core is the SOLE emitter of outbound read receipts, typing notices and presence; resolves effective policy chat > account > global; while on: receipts sent as m.read.private, typing dropped, presence withheld where applicable; manual release = signals::release_receipt(room) emitting public m.read at current position. Rule: no other module and no tauri command calls SDK receipt/typing/presence APIs directly. Per-network coupling caveats (e.g. WhatsApp) are static data shipped with risk tiers
- (decision) AD drafts: drafts table in keeper.db is source of truth (crash-safe, FR-38); mirrored debounced to per-room account data custom event type dev.keeper.draft via SDK (FR-39); conflict policy: local wins, remote surfaced for one-tap adoption. Approval Pane = cross-account query over drafts + pending outbox rows. Also write SDK Room::save_composer_draft for Element-family interop where cheap
- (decision) AD bridge manager: trait BridgeTransport with impls Provisioning (bridgev2 HTTP provisioning JSON state machine rendered as native login states) and BotDriver (programmatic bridge-bot command send/parse with timeouts; raw bot chat never hidden). Discovery pipeline merges (a) GET /_matrix/client/v3/thirdparty/protocols, (b) data-driven known-bot MXID probe registry, (c) scan of existing bot DMs/portal rooms. Health: bridgev2 bridge state events + bot ping fallback -> BridgeSession state machine (healthy/degraded/disconnected), 60s surfacing bar via notify. bbctl = optional Tauri sidecar (Apache-2.0 Go binary, bundled per-arch, launch-on-demand exec with parsed output). Network Risk Tiers + coupling caveats = versioned JSON data file in repo
- (decision) AD auth: trait AuthProvider with password, oidc (SDK OAuth + system browser + deep-link callback keeper://oauth/callback via tauri-plugin-deep-link), beeper (port of Apache-2.0 bbctl api/beeperapi flow: POST /user/login -> /user/login/email -> /user/login/response -> JWT -> org.matrix.login.jwt on matrix.beeper.com). Beeper HTTP isolated in one module with typed failure states (UI shows Beeper login unavailable); its failure can never affect non-Beeper accounts. SSS capability check gates account creation (FR-5), result logged per account
- (decision) AD notifications: notify engine in keeper-core consumes post-decryption events, applies mute/mention-only/DND rules (persisted in settings; mapped to Matrix push rules where representable, evaluated locally otherwise), posts via tauri-plugin-notification with preview toggle; click payload deep-links (account_id, room_id, event_id) -> shell focuses chat. Bridge health alerts ride same pipeline. No push infrastructure ever on desktop (NFR-11)
- (decision) AD concurrency: per-account supervision tree — AccountManager owns registry of AccountHandle{Client, SyncService tasks, archiver, signals, send scheduler} spawned on tokio; cross-account aggregators (inbox merge, palette index, approval pane, notify) consume per-account streams; UI-facing state exposed via watch/broadcast channels consumed by the shell layer. Rule: no global mutable singletons except the tauri-managed AppState handle; cross-task state only via owned tokio sync primitives
- (decision) AD inbox projection: unified inbox computed in Rust by merging N RoomListService VectorDiff streams, sorted by recency with pins hoisted, favorites sectioned, archive state and Space filter applied core-side; UI receives a windowed VM stream (visible range + buffer, total counts) via one channel; Command Palette and Quick-Switcher query a Rust in-memory chat/action index via command (100ms/10k bar). Prevents: mirroring 10k chats into JS and JS-side sorting divergence
- (decision) AD media: keeper-media://<account_id>/<media_key> custom protocol with HTTP Range support streams decrypted bytes from the SDK media cache (thumbnails via SDK thumbnail API); media blob retention configurable (default keep) independent of message-text durability.
